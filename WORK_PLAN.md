## День 1 — Каркас приложения и роутинг

1. **Скелет страниц**

- создать страницы: `CatalogPage`, `ProductPage`, `CartPage`, `LoginPage`, `ProfilePage`
- в `router` развести базовые маршруты и `not-found`

2. **Общий layout**

- `AppLayout` с header/nav/footer
- `RouterView` внутри layout + активные ссылки

3. **UI-база**

- кнопка, инпут, карточка, контейнер
- единые базовые стили/переменные

4. **Мини-итог дня**

- все маршруты открываются без ошибок
- есть структура папок `shared/entities/features/pages`

1. **API слой**

- endpoint для списка товаров

- пустое состояние, если данных нет

4. **Мини-итог дня**

## День 3 — Фильтры, сортировка, query params

1. **Локальное состояние фильтров**

- search, category, sort, page
- `computed` для derived state

2. **Синхронизация с URL**

- хранить фильтры в query params (`?search=&sort=&page=`)
- при перезагрузке восстанавливать состояние из URL

3. **Дебаунс поиска**

- `watch` + debounce 300–500ms

4. **Мини-итог дня**

- фильтры работают, URL делится ссылкой на текущее состояние

## Правило “60% сам / 40% автокомплит” (чтобы практика была честной)

- Сначала 10–15 минут пишешь сам каркас фичи.
- Автокомплит используешь только на “рутину”: типы, boilerplate, простые шаблоны.
- Все, что связано с архитектурой/данными/эффектами — сначала сам, потом сверка.
- В конце каждого дня фиксируешь: что понял, где застрял, какие вопросы принести в чат.
